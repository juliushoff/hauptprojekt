Recommended evaluation stack for the thesis Attack resistance: MCPTox + MSB (poisoning + full-pipeline). Utility regression: MCP-Universe or MCP-Bench. Report both as an NRP-style curve. For the CLI specifically, you can also build a small labelled corpus of benign vs poisoned manifests (MCP Pitfall Lab + MCPTox cases) and report precision/recall/F1 of the static detectors — that's a clean, defensible Hauptprojekt result.

Gap analysis → where your contribution sits

Honest positioning The space moved fast in 2025–2026: poisoning benchmarks (MCPTox, MSB, MCP-TDP), server scanners (MCPGuard, VIPER-MCP), and runtime guardians (MCP Guardian, IPI-Proxy) all exist. So "a scanner" or "a proxy" alone is not novel. Candidate angles that still look open:
  • Unification & coverage. Existing scanners each target one slice (taint or metadata or privilege). A single CLI that runs all four analyzer families (metadata-NL, identity, taint, supply-chain/config) against the Hou 16-threat taxonomy and reports coverage per threat is a useful, well-scoped engineering+evaluation contribution.
  • Static→runtime continuity. Few works connect the pre-flight scan to the runtime monitor. Your two artifacts sharing one threat model and one policy schema (scan emits policies the proxy enforces; proxy confirms the "both/runtime" threats the scan could only suspect) is a genuinely novel framing.
  • Rug-pull / drift detection via manifest pinning — pin-and-diff of tool manifest+code hashes bridges static and runtime and is under-explored.
  • Rigorous false-positive / utility accounting. Many attack papers report ASR but not the defense's cost. An NRP-style eval of your detectors on real servers (precision/recall + task-regression) would be a credible empirical core.
Before locking scope Confirm which of MCPGuard / VIPER-MCP / MSB have released code & datasets (MSB has a public repo) — reusing their corpora saves months and makes results comparable. Then do a focused related-work diff so the thesis claim is precisely "X exists, Y exists, but the combination/continuity/evaluation Z does not."
